05how it works3 steps · CLI demo prominent
// how it works
Copy on any device.
Paste on all of them.
Works from the terminal, editor, browser, and phone. E2E encrypted — ClipStream never sees your clipboard contents.
01 / install
One command setup
npm install -g clipstream-cli
Plus the Chrome extension and Android app. Sign in once with Google — all devices connected automatically.
02 / copy
Copy from anywhere
echo $DB_URL | cs copy
Terminal, VS Code, Chrome, or Android. ClipStream detects what you copied — code, URL, command, or secret — and labels it.
03 / paste
Paste anywhere else
cs paste | pbcopy
On your phone it appears as a notification. In VS Code, Cmd+Shift+V opens your full history. In the terminal, cs paste pipes it anywhere.
06featuresdeveloper-specific differentiators
// features
Built for the cross-platform developer stack.
Not a general clipboard tool with a developer skin. Every feature exists because of a real developer workflow problem.
🔒 Zero-knowledge encryption
Keys generated on your device. Never transmitted. ClipStream's servers only see encrypted bytes — not your clipboard contents. Ever.
⚡ Under 50ms local sync
When devices are on the same network, ClipStream syncs directly — not through a cloud relay. Faster than you can switch windows.
🧠 Content-aware detection
Knows the difference between a URL, a terminal command, a code snippet, and an API key. API keys show masked by default — in history and notifications.
💻 VS Code integration
Cmd+Shift+V opens your clipboard history inside the editor. Paste anything from any device without switching tabs.
📟 Terminal-native CLI
cs copy and cs paste work in any shell. Pipe-friendly. Shell completion for bash and zsh. Feels like a real Unix tool because it is one.
📋 90-day history
Every sync is saved and searchable. Find that connection string from last Tuesday. History is encrypted and stored per-device — never in plaintext on our servers.

Copy rules
Never write
"seamlessly" or "effortless"
"powerful" without a spec
"intuitive" (they all say this)
"cutting-edge" or "next-gen"
"leverage" as a verb
"solution" for a specific tool
Any sentence starting with "We believe"
Always write
Specific numbers ("under 50ms", "90-day")
Second person ("you", "your")
What it does before what it is
The uncomfortable truth ("emailing API keys")
Short sentences. Then shorter.
Tech terms without apologising for them
What it doesn't do (builds trust)
