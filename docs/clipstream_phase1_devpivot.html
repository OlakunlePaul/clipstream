Phase 1 · Developer pivot
Revised tech stack
Reoriented around the cross-OS developer workflow. Three new components added: CLI tool, VS Code extension, and content-aware detection. Electron dropped; Supabase + Ably retained as the relay backbone.
Full stack — Phase 1
Chrome extension
TypeScript + Vite + MV3
Build pipeline, type safety, MV3 offscreen document for background clipboard monitoring
CLI tool new
Node.js + TypeScript
npm package — cs copy / cs paste. Pipes into the same Ably sync channel. Works on macOS, Linux, Windows
VS Code ext new
vscode Extension API
Command palette: "Paste from ClipStream". Reads history from shared Supabase session, no separate auth
Android app
React Native + Expo
Android first. iOS added Phase 2. expo-clipboard, expo-secure-store for key storage
Auth
Supabase Auth
Google sign-in via Chrome identity API. CLI uses device auth flow (browser pop-up once)
Realtime relay
Ably
WebSocket pub/sub across all surfaces. Local LAN sync added Phase 2 via WebRTC/mDNS
Encryption
tweetnacl
Works in browser, Node, and React Native. Zero-knowledge: private keys never leave the device
Database
Supabase Postgres
devices, clip_events, users, pinned_snippets tables. RLS on all tables
Backend
Supabase Edge Functions
Deno. Auth exchange, device registration, rate limiting, content-type detection
Content detection new
Custom classifier
Regex + heuristics: detects code, URLs, commands, API keys, file paths. Runs client-side only
Monitoring
Sentry
Error tracking across extension, CLI, VS Code ext, and Android. Configured to never log clipboard content
Distribution
npm + Chrome Web Store + VS Code Marketplace
Three publish targets from one monorepo. CLI on npm, extensions on their respective stores
Build order + dependencies
Sequence
1.
Encryption layer — shared lib used by all surfaces. Build and test first.
2.
Backend + DB schema — devices, auth, rate limiting. Everything else depends on this.
3.
Chrome extension — primary surface. Most complex MV3 offscreen logic lives here.
4.
CLI tool — reuses encryption lib and Ably client. Fastest to build after extension.
5.
Android app — reuses encryption lib. Parallel with CLI if 2 engineers.
6.
VS Code extension — reads from Supabase; no Ably needed. Lowest complexity.
7.
Quality + monitoring — add Sentry across all surfaces before first user.
What's changed from original plan
+
CLI tool added — terminal-native devs are the core audience; this is table stakes
+
VS Code extension added — makes ClipStream sticky; replaces Electron in value
+
Content detection added — developer-specific: code, API keys, file paths, commands
+
API key masking added — security angle is a core differentiator, not a feature
–
Electron dropped — CLI + VS Code ext replaces the desktop app for dev workflow
–
Custom auth dropped — Chrome identity + Supabase auth handles it without the overhead
Monorepo structure: Put all surfaces in one repo — packages/shared-crypto, packages/shared-ably, packages/chrome-ext, packages/cli, packages/vscode-ext, packages/android. Turborepo or pnpm workspaces handles the build graph. The shared packages are what make the CLI and VS Code ext fast to build — they reuse the same encryption and sync logic as the extension.

Exports: generateKeyPair(): {publicKey: Uint8Array, privateKey: Uint8Array} exportKey(key: Uint8Array): string — base64 encode for storage/transport importKey(str: string): Uint8Array — base64 decode encryptForDevice( plaintext: string, theirPublicKey: string, — base64 encoded myPrivateKey: string — base64 encoded ): {ciphertext: string, nonce: string} — both base64 decryptFromDevice( ciphertext: string, — base64 nonce: string, — base64 theirPublicKey: string, — base64 (sender's public key) myPrivateKey: string — base64 ): string — plaintext, throws on failure encryptForMultipleDevices( plaintext: string, devices: Array<{deviceId: string, publicKey: string}>, myPrivateKey: string ): Array<{deviceId: string, ciphertext: string, nonce: string}> Add Vitest unit tests: 1. Backend ▶
Prompt
Create the Supabase database schema for ClipStream. Store all in /supabase/migrations/. Tables: users: id uuid PK (refs auth.users) email text plan text default 'free' -- 'free' | 'pro' | 'team' syncs_today int default 0 syncs_reset_at timestamptz default now() created_at timestamptz default now() devices: id uuid PK default gen_random_uuid() user_id uuid FK → users.id public_key text NOT NULL device_name text platform text -- 'chrome-ext' | 'cli' | 'vscode' | 'android' | 'ios' last_seen timestamptz default now() created_at timestamptz default now() UNIQUE(user_id, platform, device_name) clip_events (metadata only — NO plaintext, NO ciphertext): id uuid PK default gen_random_uuid() user_id uuid FK → users.id sender_device_id uuid FK → devices.id content_type text -- CODE | COMMAND | URL | API_KEY | FILE_PATH | ERROR_TRACE | TEXT is_sensitive boolean default false content_length int recipient_count int ts timestamptz default now() RLS policies: - users: SELECT/UPDATE own row only (auth.uid() = id) - devices: SELECT all where user_id = auth.uid(), INSERT/UPDATE own device only - clip_events: INSERT where user_id = auth.uid(), SELECT last 7 days where user_id = auth.uid() pg_cron job: reset syncs_today to 0 at midnight UTC daily. Backend ▶
Prompt
Build four Supabase Edge Functions (Deno) for ClipStream.
